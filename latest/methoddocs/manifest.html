Update compilers in the manifest. This method appends any given compiler with a a different name, version, and settings combination.

Parameters:

compilers (List[~ethpm_types.source.Compiler]) – A list of compilers.

compilers: list[ethpm_types.source.Compiler] | None

Information about the compilers and their settings that have been used to generate the various contractTypes included in this release.

contract_types: dict[str, ethpm_types.contract_type.ContractType] | None

ContractType objects that have been included in this release.

  • Should only include types that can be found in the sources.

  • Should not include types from dependencies.

  • Should not include abstracts.

dependencies: dict[ethpm_types.manifest.PackageName, Union[pydantic.networks.FileUrl, pydantic.networks.AnyUrl]] | None

A mapping of EthPM packages that this project depends on. The values must be content-addressable URIs that conforms to the same manifest version as manifest.

deployments: dict[eth_pydantic_types.bip122.Bip122Uri, dict[str, ethpm_types.contract_type.ContractInstance]] | None

Information for the chains on which this release has ContractInstance references as well as the ContractType definitions and other deployment details for those deployed contract instances. The set of chains defined by the BIP122 URI keys for this object must be unique. There cannot be two different URI keys in a deployments field representing the same blockchain. The value of the URIs is a dictionary mapping the contract instance names to the instance themselves. The contract instance names must be unique across all other contract instances for the given chain.
